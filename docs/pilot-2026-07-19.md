# Pilot report — greenfield todo app (C13)

Date: 2026-07-19 · Harness: PI (not Claude Code) · Models: MiniMax M3
(8 turns, dropped for connection issues) → GLM 5.2 (remainder) ·
Skills: user-level `~/.agents/skills` (open-standard discovery) ·
Repo: ~/Developer/craftsman-pilot ("One Thing", Svelte 5 + Vite + Dexie,
bun-only, local-first).

## Verdict

C13 holds. From the single prompt "build a todo app", the flow
auto-routed end to end — brainstorm → craftsman-mode init → specify
(human freeze) → plan → implement/verify per task → review per batch →
fix(review) → boundary re-plan × 5 batches → finish — with **zero
process steering**: of 12 human messages, 9 are product-owner content
(the opening prompt, brief answers, approvals, "We always use bun!"),
1 a defect report, 2 bare "continue" after stream drops. No skill was
ever named.

Shipped: 16/16 spec criteria, 44 commits, 5 reviewed batches.
Gates re-verified independently after the run: 53/53 tests, eslint
clean, svelte-check 0 errors, build + SW precache green.

Flow ceremony (everything except implement) ≈ 53% of turns, ≈ 55% of
cost — it bought 5 review passes (6 findings, all fixed in typed
commits), 4 ADR entries with rejected alternatives incl. 2 dead ends,
honest Gaps, and a git log a human can audit commit by commit.

## What held on a mid-tier model

- Red→green TDD per task, "red for the right reason" checked aloud.
- Refused early review: "not all verified yet, so we don't route".
- C3 edge exercised: no subagent on PI → declared degraded self-review,
  which still produced real findings (a11y labels, stale preventDefault).
- Discovery routes honored: pre-existing bug logged to Gaps, not
  drive-by fixed; product questions deferred to the human.
- Typed commits with C-ids throughout; verify commits tick boxes.

## Friction — findings for the improvement batch

1. **Constitution budget breached silently (C1/C10).** Init wrote a
   161-line AGENTS.md (≤100 promised). Always-on context ≈ 2.8k tokens
   — over C10's 2k bound — purely from the oversize. Nothing mechanical
   catches it: craftsman-mode needs a line-count gate at init, and this
   repo's check.sh should pin the template as rendered constitution
   (already an open Gap).
2. **Open defect lost to a stream drop (C9 doctrine, mid-session).**
   White-screen report routed to fix correctly, diagnose-first began —
   then the stream died mid-hypothesis; "Continue" resumed the previous
   implement thread and the defect was never mentioned again (likely
   benign — missing empty state, implemented later in 3.3 — but never
   verified or closed). The defect lived only in conversation context.
   Fix skill should ledger the defect (PLAN State/Gaps) as its first
   step, before diagnosing — compaction-survival doctrine applied to
   interruption.
